
# Conectando Futuros 🌐

> Plataforma digital integrada para a inclusão sustentável de imigrantes no mercado de trabalho brasileiro.

O **Conectando Futuros** é uma solução de software desenvolvida para mitigar barreiras linguísticas, burocráticas e geográficas encontradas por imigrantes no Brasil (com foco inicial na região de Chapecó-SC). O projeto une pesquisa socioeconômica, engenharia de requisitos e desenvolvimento de software web para promover uma inserção laboral digna e justa.

Este projeto foi premiado e apresentado na **FECITI (Feira de Ciência, Tecnologia e Inovação)**, e na **FEICITEC ( Feira Estadual de Ciências e Tecnologia)** tendo como base metodológica uma pesquisa de campo quantitativa e qualitativa estruturada.
## ⚖️  Status do Projeto (MVP)

### 🎨 Prototipação de Alta Fidelidade (Front-End)
Este ecossistema foi desenvolvido utilizando estritamente **HTML5 e CSS3**, operando como um **Protótipo de Alta Fidelidade / MVP (Minimum Viable Product)**. 

A escolha de focar exclusivamente na camada de apresentação (Front-End) e na arquitetura de fluxos visuais fez parte da estratégia inicial. O objetivo central foi validar a usabilidade, a acessibilidade linguística e a aderência das interfaces junto aos usuários e empresas parceiras na feira tecnológica, eliminando a complexidade imediata de infraestrutura de back-end ou banco de dados para priorizar a experiência e a validação dos requisitos de negócio.

### 📉 Análise Demográfica e Ciclo de Vida do Software
O escopo inicial do projeto foi fundamentado no intenso fluxo migratório de venezuelanos e haitianos para a região de Chapecó-SC, mapeado durante a nossa pesquisa de campo. 

Contudo, alinhado às práticas reais de governança de produto, o desenvolvimento ativo da plataforma foi pausado  em sua **versão estável de portfólio** devido a dois fatores de mercado:
1. **Mudança do Fluxo Geopolítico:** Os indicadores demográficos locais apontaram uma estabilização e subsequente desaceleração drástica na chegada de novas ondas migratórias dessas nacionalidades específicas para a região.
2. **Absorção de Mercado:** O público-alvo mapeado inicialmente foi, em grande parte, integrado ou redirecionado, alterando a urgência e o volume da dor de mercado que o software visava mitigar.

Dessa forma, o projeto cumpre com excelência o seu papel como **estudo de caso científico**, validação de engenharia de requisitos e demonstrativo de competência em arquitetura de interfaces estruturadas.

## 🏗️ Arquitetura e Estrutura do Projeto

Para garantir a viabilidade e o alinhamento com as necessidades reais dos usuários, o desenvolvimento foi dividido em três pilares fundamentais:

1. **Pesquisa Científica & Validação:** Levantamento de dados com a comunidade imigrante para mapear os principais gargalos (validação de diplomas, barreira idiomática e acesso a vagas formais).
2. **Engenharia de Requisitos:** Mapeamento detalhado de prioridades (Alta, Média, Baixa) e matriz de responsabilidades para garantir a usabilidade e o valor de entrega da plataforma.
3. **Desenvolvimento de Software:** Implementação de uma interface  web responsiva focada em acessibilidade e facilidade de uso.

* **Construtor de Currículos Adaptivo:** Mecanismo projetado para auxiliar o imigrante na estruturação de suas experiências profissionais de forma simples e intuitiva para o padrão de contratação brasileiro.
